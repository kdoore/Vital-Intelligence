# SR2 GitBook Refinement Protocol

### Publishing Contract (Two Homes)

* GitBook = Public / Teach (curated, stable, reusable)
* Dropbox = Working / Archive (raw assets, drafts, exports) Rule: GitBook may link to Dropbox assets, but Dropbox does not define concepts.

### Canonical Anchors (definitions live here)

1. SR2\_README (orientation + rules)
2. Bridging Spiral & VIM Pillars (with Recursion X)
3. Dashboard Dials (instrument panel)
4. Episode Card Template (unit of iteration) Optional: Narrative Engine (only when active this sprint)

Rule: non-anchor pages may not redefine core terms; they link to anchors.

### Page Triage Pass (once per sprint)

For every GitBook page, record:

* learner job (one sentence)
* primary anchor dependency
* status: Keep / Merge / Split / Archive
* vehicle: Card game / Teaching case / Theory / Tooling

### Page Refactor Recipe (apply to every kept page)

1. Purpose (2–3 lines)
2. When to use (1 line)
3. Anchor links (2–4 links)
4. One usable artifact (template/checklist/example)
5. Footer: Version + last updated + what changed

### Vocabulary Standard (project-wide)

* Recursion (X) = metacognitive loop (how confidence formed)
* Integration = the outcome of learning (what gets incorporated)
* Resilience = ability to return to learning under oscillation Rule: if older pages use “Integration” as a step, update to “Recursion (X) → Integration.”

### Sprint Constraint (anti-mess)

Only one prototype vehicle is “active” per sprint. All other vehicles remain parked unless a Decision Record explicitly activates them.
