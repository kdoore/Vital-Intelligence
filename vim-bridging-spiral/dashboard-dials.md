# Dashboard Dials

## Dashboard Dials: Aperture + MPCM + Bridging Suits

This page is the **instrument panel** for the Bridging Spiral. Use it as a fast reference during learning episodes, teaching cases, or daily media use.

**Core premise:** meaning does _not_ arrive fully formed. In studio terms, meaning **emerges** through **Material → Process → Context → Meaning (MPCM)** across time. In AI-mediated environments, _Material/Process/Context are often hidden_, so users absorb shallow frames without consent.

***

### Two interaction modes (choose the right instrument posture)

#### Mode 0 — Drift (default)

Entertainment / distraction / social scrolling.\
**Risk:** you still absorb **frames** (what’s normal, who’s “good/bad,” what’s urgent) and your nervous system updates “autopilot” without explicit learning intent.

#### Mode 1 — Inquiry (chosen)

Learn / decide / verify / create.\
**Benefit:** you explicitly widen aperture, slow premature convergence, and choose prosocial action with better boundaries.

**Bridge principle:** we do _not_ demand constant analysis. We install **tiny interventions** so Drift doesn’t become harm.

***

### MPCM Overlay (make hidden layers visible)

Use this overlay anytime you feel pulled, confused, or activated.

* **M — Material:** what is this _made of_? (text, image, voice, feed, AI summary, interface)
* **P — Process:** what _generated/amplified_ it? (training/ranking, persuasion loops, editing, automation, virality)
* **C — Context:** what surrounds it? (incentives, audience, timing, your state, cultural pressure)
* **M — Meaning (emergent):** what is forming in me? (emotion, assumption, normalization, identity posture)

**Discernment = adding missing M/P/C so meaning doesn’t get hijacked by shallow frames.**

***

### How to use the dashboard (30–90 seconds)

1. Check **Aperture (A)** + **♠ Spades** first. If aperture is narrow, do not force conclusions.
2. Run **♦ Diamonds** to name frames/claims, calibrate confidence (T/I/F), compare models, and include incentives/externalities.
3. Use **♥ Hearts** to choose a prosocial action, scale verification, and set consent-based boundaries.
4. Use **♣ Clubs** to loop back when captured and to integrate learning over time (sleep/iteration/analog practice).
5. Use **❄ Kindness** to widen aperture and protect repair capacity across scales.

***

## A — Aperture (master dial)

**A = bandwidth to perceive + integrate + choose right now.**\
Aperture determines how wide your model-space is and how quickly you collapse uncertainty into a story.

#### A has three faces (map to suits)

* **Aₛ (♠ Somatic):** body bandwidth (can I sense and settle?)
* **A𝚌 (♦ Cognitive):** hypothesis bandwidth (can I hold more than one model?)
* **Aᵣ (♥ Relational):** co-regulation bandwidth (can I include other minds without threat?)

**When A is low:** tunnel vision, compulsive clicking, urgency, contempt for uncertainty, story lock-in.\
**Rule:** if A is low, _do not force convergence._

#### Dose ladder (choose the smallest effective widening move)

* **Micro (60–120s):** orient + breathe + stand + change posture/light
* **Meso (10–20m):** walk outside / analog mark-making / slow movement
* **Macro (sleep):** consolidation (autopilot shaping)

***

## ♠ Spades — Somatic Gyroscope (Sensing + Stabilizing)

### ♠ PI — Pause & Intention

**Function:** install consent before being pulled: purpose, scope, time boundary. **When low:** autopilot drift, “just one more,” reactive responding. **Micro-action:** 1-sentence intention + timer + exit condition.

### ♠ WT — Window of Tolerance (State)

**Function:** track activation/fatigue/threat that gates learning. **When low:** fight/flight/freeze/fawn patterns; numbness; collapse. **Micro-action:** regulate first; shrink stakes; switch to a reversible move (draft, save, step away).

### ♠ FE — Felt Expectations (Active Inference Priors)

**Function:** notice “body predictions” shaping perception before interpretation. **When low:** “this feels true” becomes “this is true.” **Micro-action:** name one felt expectation:

> “My system expects \_\_\_ (danger/rejection/urgency).”\
> Then choose **Micro/Meso/Macro** dose to widen Aₛ.

***

## ♦ Diamonds — Cognitive Radar (Discernment + Modeling)

### ♦ FC — Frame/Claim Scan (lightweight)

**Function:** notice what lens is being installed _and_ what is being asserted.

* **Frame** = the lens (threat/status/outrage/helplessness/savior narrative)
* **Claim** = the assertion inside the frame (explicit or implied) **Micro-action (5–10s):** label one frame + one claim type (norm/identity/causal/authority/permission).

### ♦ CC — Confidence Calibration (Neutrosophic T/I/F)

**Function:** keep uncertainty explicit; prevent premature convergence.

* **T** supported
* **I** indeterminate
* **F** unsupported **Micro-action:** mark one key point as **I** before sharing/escalating.

### ♦ MM — Multi-Model Compare

**Function:** widen hypothesis space under VUCA. **When low:** single narrative lock-in. **Micro-action:** write one alternative model:

> “Another explanation could be \_\_\_.”

### ♦ IE — Incentives & Externalities

**Function:** include the shadow field.

* Incentives: platform/actor benefits + your internal relief-seeking
* Externalities: local internal (anxiety/compulsion), local social (conflict), non-local (ecology/labor/civic trust) **Micro-action:** name **1 incentive + 1 externality**.

***

## ♥ Hearts — Relational Prosocial Compass (Values + Verification + Repair)

### ♥ PV — Prosocial Values Field (across space/time)

**Function:** orient by care/dignity/non-harm across self → group → society → ecology → future. **Micro-action:** “Who is vulnerable here?” + “What reduces harm?”

### ♥ VS — Verification Scale / Reality Check

**Function:** match rigor to stakes; reintroduce external reality when needed.

* **Low stakes:** note uncertainty; don’t amplify
* **Medium:** check one independent source
* **High:** pause + human/institution reality check **Micro-action:** choose the scale _before_ escalating.

### ♥ CR — Consent & Repair (Boundaries + Repair Move)

**Function:** protect autonomy and relational safety; prevent boundary dissolution. **Micro-action:** choose one:

* boundary: “I’m done for now.” / “I’ll revisit tomorrow.” / “Not engaging this thread.”
* repair: apologize, clarify, de-escalate, reconnect to a trusted person

***

## ♣ Clubs — Recursion & Integration (Across time)

### ♣ RL — Recalibration Loop

**Function:** loop back when you feel captured (urgency, outrage, flattery, isolation cues, authority illusion). **Micro-action (60–120s):**

* 1 confidence cue (why I feel sure)
* 1 missing variable
* 1 update condition (what would change my mind)

### ♣ MU — Model Update

**Function:** make learning explicit (not just reaction). **Micro-action:** write one update:

* new boundary rule / new question / new constraint / new practice

### ♣ Cn — Consolidation (Sleep / Time / Analog Integration)

**Function:** integration takes time; autopilot forms through repetition + consolidation. **Micro-action:** decide what gets deferred to sleep/time; set a reminder.

***

## ❄ KR — Kindness Resonance (cross-cut dial)

**Function:** regulated prosocial capacity that widens aperture and supports repair across scales.\
Not “niceness”: warm truth + firm limits + commitment to repair.

**Micro-action:** the smallest move that reduces harm _and_ preserves learning.

***

### Fast notation for SR2 logs

Use 0–10 or Low/Med/High.

Example: A:6 (Aₛ:5 A𝚌:7 Aᵣ:6) / ♠PI:7 ♠WT:5 ♠FE:6 / ♦FC:6 ♦CC:5 ♦MM:4 ♦IE:6 / ♥PV:7 ♥VS:6 ♥CR:7 / ♣RL:5 ♣MU:6 ♣Cn:4 / ❄KR:8

**Limiter rule:** name the one dial that most constrained the episode, then choose one action that raises it by one notch.
